# Методология тестирования математических функций

Реализовать интервальный подход к тестированию корректности для ранее реализованной функции `logf` в указанном интервале.

-  В качестве референса можно использовать `double-precision` функцию `log(x)` библиотеки `math.h`, либо `MPFR` с достаточной точностью или аналоги.
- Поставить ограничение точности в `4.0 ULP`. Можно использовать определение `ULP`, данное ранее в лекции.
- Реализовать интервальное тестирование, выбрав интервалы таким образом, чтобы они приблизительно равномерно покрывали множество чисел с плавающей точкой. В каждом интервале достаточно взять `~1000-10000` точек.
- Протестировать функцию на специальные значения.
- Дополнительно протестировать функцию на флаги и `errno`. Разные библиотеки могут выставлять `errno` по-разному, ориентироваться можно на `libc libm`.

<br>

## Методология тестирования

### 1. Оценка погрешности (ULP)
В проекте используется математическое определение ULP (Unit in the Last Place), изложенное в материалах курса:
$$ulp_{error} = \frac{|a - A|}{2^{exp(A) - 23}},$$
где:
* $a$ — вычисленное значение (`float`).
* $A$ — эталонное значение (`double` из `math.h`).
* $23$ — ширина мантиссы для формата `binary32`.

### 2. Стратегия покрытия
Тестирование разделено на три этапа:

* **Edge Case Testing**: Проверка поведения в особых точках.
    * *Отрицательные числа и $-\infty$*: проверка на `NaN`, `errno = EDOM` и флаг `FE_INVALID`.
    * *Нули ($\pm 0.0$)*: проверка на $-\infty$, `errno = ERANGE` и флаг `FE_DIVBYZERO`.
    * *Единица ($1.0$)*: проверка на получение точного нуля без побочных флагов.
* **Precision Testing**: Точечная проверка на представительном наборе данных (нормализованные, субнормальные числа и значения вблизи границ точности).
* **Interval Testing**: Равномерное покрытие всей области определения. Весь диапазон экспонент разделен на интервалы, внутри которых проверяется по 10,000 случайных точек для статистического подтверждения максимальной погрешности.

## Структура проекта
* `src/` — исходный код тестов (`main.c`, `tests.c`) и визуализация таблицы (`graphics.c`).
* `include/` — заголовочные файлы.
* `lib/` — объектный файл тестируемой функции (`logf.o`).
* `bin/` — скомпилированный исполняемый файл.

## Сборка и запуск

### Требования
* Компилятор `gcc`
* Стандартная библиотека `math` (флаг `-lm`)

### Сборка проекта:
```bash
make
```

### Запуск тестов:
```bash
./bin/test_runner
```
